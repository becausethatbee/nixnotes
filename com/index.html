Установка Docker на CentOS Stream 10¶

Ниже приведён полный скрипт с пояснениями в комментариях. Его можно использовать как пошаговую инструкцию — просто выполняйте команды по порядку.

```bash

================================¶

1. Обновляем систему¶

dnf — менеджер пакетов CentOS Stream 10¶

update — обновление всех установленных пакетов¶

-y — автоматическое подтверждение¶

Перед установкой нового ПО рекомендуется обновить систему,¶

чтобы избежать конфликтов зависимостей.¶

sudo dnf update -y

================================¶

2. Устанавливаем необходимые плагины dnf¶

dnf-plugins-core — набор расширений для dnf, включая config-manager¶

Эти плагины обязательны для добавления репозиториев.¶

sudo dnf install -y dnf-plugins-core

================================¶

3. Подключаем официальный репозиторий Docker¶

--add-repo — флаг добавления нового репозитория¶

Используем официальный репозиторий для гарантии актуальности и безопасности пакетов.¶

sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

================================¶

4. Устанавливаем Docker и сопутствующие пакеты¶

docker-ce — движок Docker (Community Edition)¶

docker-ce-cli — клиент командной строки¶

containerd.io — контейнерный рантайм¶

docker-buildx-plugin — расширенные возможности сборки образов¶

docker-compose-plugin — встроенный docker compose¶

Рекомендуется устанавливать все эти пакеты сразу,¶

чтобы не тратить время на доустановку позже.¶

sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

================================¶

5. Запускаем и включаем автозапуск Docker¶

enable — добавляет сервис в автозагрузку¶

--now — сразу запускает его¶

Это позволит Docker работать автоматически после каждой перезагрузки.¶

sudo systemctl enable --now docker

================================¶

6. Добавляем пользователя в группу Docker (по желанию)¶

-aG — добавить пользователя в группу без удаления из других¶

$USER — переменная с именем текущего пользователя¶

Это позволит использовать Docker без sudo.¶

Важно: доступ к Docker фактически равен правам root —¶

применяйте это только на доверенных системах.¶

sudo usermod -aG docker $USER

Применяем изменения групп в текущей сессии¶

newgrp docker

================================¶

7. Проверяем установку¶

docker version — проверка версий клиента и сервера¶

docker compose version — проверка наличия плагина compose¶

docker run --rm hello-world — запуск тестового контейнера¶

--rm — автоматически удалить контейнер после завершения¶

Если вывод hello-world сообщает, что установка прошла успешно —¶

Docker готов к работе.¶

docker version docker compose version docker run --rm hello-world
